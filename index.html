Full Stack Apps

Description

Designed and developed a full-stack internal application to centralize, manage, and visualize all corporate entities within the organization.

Problem

The organization lacked a reliable way to accurately represent, track, and analyze a growing number of legal entities; information was fragmented, difficult to update, and not easily searchable or visualizable.

Solution

Drive Scripts

Description

Developed a script-based automation to generate NDAs dynamically based on user input, ensuring correct template selection, naming conventions, and file storage.

Problem

NDA creation was a repetitive, manual process requiring multiple steps (template selection, data insertion, file naming, and folder organization), increasing time spent and risk of errors.

Solution

Built an automated workflow that collects key inputs, selects the appropriate NDA template, populates updated information, applies standardized naming, and saves the document in the correct directory, significantly reducing manual effort and turnaround time.

Chrome extensions

Description

Designed and built a Chrome extension to enhance Jira ticket workflows by surfacing critical legal information and guiding correct document handling.

Problem

A high volume of Jira tickets required manual review due to recurring issues such as missing information, incorrectly named documents, and conditions requiring legal pushback, resulting in slower workflows and increased risk of errors.

Solution

Developed a custom Chrome extension that flags red issues directly within Jira tickets, highlights missing or critical information, suggests standardized document naming conventions, and surfaces counterparty details to streamline document preparation and review.

Zapier

Description

(IN DEVELOPMENT) End-to-end automation project focused on streamlining contract envelope creation, signature execution, and post-signature handling by integrating structured inputs, e-signature workflows, and Jira ticketing.

Problem

Envelope creation is a largely manual and time-consuming process, taking up to an hour per paralegal each day. Tasks include manually entering recipients, CCs, signatories, and signature placement, as well as tracking fully executed agreements and uploading them to Jira, causing tickets to remain open longer than necessary.

Solution

Designing an automated workflow that generates and validates draft envelopes from structured inputs, sends them automatically for signature, and uploads executed agreements back into Jira upon completion.
